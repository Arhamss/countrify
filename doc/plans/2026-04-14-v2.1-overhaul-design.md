# Countrify v2.1 Overhaul Design

**Date:** 2026-04-14
**Status:** Approved
**Scope:** Architecture cleanup, API consistency, accessibility, performance, testing

---

## 1. Architecture & File Structure

### Remove Legacy Code
- Delete old `CountryPicker` widget from `country_picker.dart` (includes duplicate `CountryPickerTheme` and `CountryPickerConfig` classes)
- `ComprehensiveCountryPicker` becomes the new `CountryPicker`
- Remove `ModalCountryPicker` and `ModalComprehensivePicker` (consolidated into picker modes)

## 2. API Cleanup & Consistency

### Enum Unification

Replace `PickerDisplayType` and `PickerOpenType` with:

```dart
enum CountryPickerMode {
  dropdown,
  bottomSheet,
  dialog,
  fullScreen,
  none,
}
```

Parameter name: `pickerMode` on all widgets.

### Callback Naming

| Widget | Before | After |
|--------|--------|-------|
| PhoneNumberField | `onPhoneNumberChanged` | `onChanged` |
| PhoneNumberField | `onCountryChanged` | `onCountryChanged` (stays) |
| CountryDropdownField | `onCountrySelected` | `onChanged` |
| CountryDropdownField | `onCountryChanged` | removed (duplicate) |
| CountryPicker | `onCountrySelected` | `onChanged` |

### Model Fix

Add missing `largestCity` parameter to `Country.copyWith()`.

### `CountrifyFieldStyle` Addition

`focusedFillColor` — already implemented. The fill color when the field has focus (keyboard open). Falls back to `fillColor` when null.

## 3. Accessibility

### Semantics on All Interactive Elements

- **CountryFlag**: `Semantics(label: 'Flag of ${country.name}')`
- **Country list items**: `Semantics(label: '${country.name}, dial code +${code}')`
- **Search field**: `Semantics(label: 'Search countries')`
- **Dropdown icon**: `Semantics(label: 'Open country picker')`
- **Close/clear icons**: `Tooltip` + `Semantics` on all icon buttons

### Shared `CountryFlag` Widget (Public API)

```dart
class CountryFlag extends StatelessWidget {
  const CountryFlag({
    required this.country,
    this.size = const Size(24, 18),
    this.borderRadius = const BorderRadius.all(Radius.circular(4)),
    this.borderColor,
    this.borderWidth = 0.5,
    this.emojiTextStyle,
  });
}
```

Replaces 3+ duplicated flag implementations. Exported for standalone use.

### Other Shared Widgets (Public API)

- **`CountrySearchBar`** — theme-aware search field with debounce
- **`CountryListView`** — filtered list with selection, empty state
- **`CountryListTile`** — country row with flag, name, dial code, checkmark

## 4. L10n Split & Performance

### Locale File Split

`country_name_l10n.dart` (33,559 lines) split into:
- Registry class with lazy-loading cache
- 132 individual per-locale files

```dart
class CountryNameL10n {
  static final _cache = <String, Map<String, String>>{};

  static Map<String, String> getTranslations(String locale) {
    return _cache[locale] ??= _loadLocale(locale);
  }
}
```

### Performance Improvements

- Debounce on `PhoneNumberField` input (200ms, matching search pattern)
- `const` constructors on `CountryPickerTheme`, `CountrifyFieldStyle`, `CountryPickerConfig`
- `itemExtent` on all `ListView` instances

### Lightweight Phone Validation (Zero-Dependency)

- `PhoneMetadata` data class: `minLength`, `maxLength`, `exampleNumber` per country
- Baked into `all_countries.dart`
- `PhoneNumberField` auto-wires:
  - Basic length validation (if no custom `validator` provided)
  - Hint text from example number
  - `maxLength` from metadata

## Breaking Changes Summary

- `CountryPicker` (old base widget) removed
- `ComprehensiveCountryPicker` renamed to `CountryPicker`
- `ModalCountryPicker` / `ModalComprehensivePicker` removed (use `CountryPickerMode`)
- `PickerOpenType` / `PickerDisplayType` replaced with `CountryPickerMode`
- `onCountrySelected` renamed to `onChanged`
- `onPhoneNumberChanged` renamed to `onChanged`
- `pickerType` parameter renamed to `pickerMode`
